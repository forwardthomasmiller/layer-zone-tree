## Key Concepts

### Layer (å±‚)
Divide code by **technical responsibility**:
- **UI Layer** â€” Everything users see and interact with
- **Logic Layer** â€” Business rules and domain logic
- **Data Layer** â€” Data persistence and access

### Zone (åŒº)
Group files by **business responsibility** within each layer:
- **TaskZone** â€” All task-related logic
- **TimerZone** â€” All timing functionality
- **ScheduleZone** â€” All scheduling features

### Tree (æ ‘)
A "panoramic photo" of your entire project:
- Shows all Layers and Zones
- Maps file ownership
- Reveals dependencies
- Spots violations

## The CLVZ Framework

**Layer-Zone Tree** is part of the larger **CLVZ** framework:

- **C** â€” Contract-first (define interfaces between Zones)
- **L** â€” Layered (organize by technical responsibility)
- **V** â€” Validated (ensure rules are followed)
- **Z** â€” Zoned (organize within layers by business domain)

This book focuses on **L** and **Z**. The Tree makes them visible.
